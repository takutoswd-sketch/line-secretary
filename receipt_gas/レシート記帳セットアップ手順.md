# レシート記帳 セットアップ手順（約5分）

LINEでレシート写真を送ると、Googleスプレッドシートに自動記帳され、画像はGoogle Driveに保存される仕組み。

## 仕組み

```
LINEでレシート撮影・送信
  → Railwayサーバーが受信
  → Claude Visionが日付・店名・金額・勘定科目を読み取り
  → Googleスプレッドシートに1行追記 + 画像をDriveに保存
  → LINEに「✅記帳しました」と返信
```

## 手順

### 1. スプレッドシート作成
1. https://sheets.new を開く
2. 名前を「経費帳2026」などに変更（名前は自由）

### 2. Apps Script設定
1. メニュー「拡張機能」→「Apps Script」
2. 開いたエディタの中身を全部消して、`Code.gs` の内容を貼り付け
3. 保存（Cmd+S）

### 3. Webアプリとしてデプロイ
1. 右上「デプロイ」→「新しいデプロイ」
2. 歯車アイコン →「ウェブアプリ」を選択
3. 設定:
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**
4. 「デプロイ」→ Googleアカウントの承認を求められたら許可
   （「安全ではないページ」警告が出たら「詳細」→「移動」で進む。自分のスクリプトなので問題なし）
5. 表示される **ウェブアプリURL**（`https://script.google.com/macros/s/xxx/exec`）をコピー

### 4. Railwayに環境変数を追加
1. https://railway.com/project/79fd8803-afbd-42b8-aa11-81b26a0f42c5 を開く
2. サービスの「Variables」タブ
3. 追加: `GAS_WEBHOOK_URL` = （手順3でコピーしたURL）
4. 再デプロイが自動で走る（GitHubへのpushでも再デプロイされる）

### 5. テスト
LINEでレシート写真を1枚送信 → 「✅記帳しました」が返ってくればOK。
スプレッドシートに行が追加され、Driveに「レシート画像/2026」フォルダができているはず。

## スプレッドシートの列

| 日付 | 店名 | 金額(税込) | うち消費税 | 勘定科目 | 支払方法 | 摘要 | 読取精度 | 画像リンク | 登録日時 |

- 勘定科目はAIが自動判定（消耗品費・旅費交通費・接待交際費など）。間違っていたらシート上で直せばOK
- 読取精度が「low」の行は金額・日付を目視確認推奨
- 画像リンクから元のレシート画像を確認可能（電子帳簿保存法の保存要件対策）

## トラブルシューティング

- **「⚠️ スプレッドシート未設定」と返ってくる** → 手順4のGAS_WEBHOOK_URLが未設定
- **「レシートとして認識できませんでした」** → 明るい場所で全体が写るように撮り直す
- **GASコードを修正したとき** → 「デプロイ」→「デプロイを管理」→ 鉛筆アイコン → バージョン「新バージョン」→ デプロイ（URLは変わらない）
